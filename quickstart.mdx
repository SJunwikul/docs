## Get started with ARES

This guide will walk you through setting up the ARES TypeScript SDK and making your first API call.

<Note>
  **Prerequisites**: You need an ARES partner account. Visit
  [joinares.com](https://joinares.com) to register as a partner and obtain your
  credentials.
</Note>

## Step 1: Install the SDK

Install the ARES SDK via npm, yarn, or pnpm:

<CodeGroup>

```bash npm
npm install ares-sdk
```

```bash yarn
yarn add ares-sdk
```

```bash pnpm
pnpm add ares-sdk
```

</CodeGroup>

## Step 2: Set up environment variables

Create a `.env` file in your project root and add your ARES credentials:

```bash .env
ARES_CLIENT_ID=your_partner_client_id
ARES_CLIENT_SECRET=your_partner_client_secret
```

<Warning>
  Never commit your `.env` file or expose credentials in client-side code. The
  ARES SDK is server-side only.
</Warning>

## Step 3: Set up authentication

Choose your authentication method (guides for other frameworks are [here](/authentication/overview#integration-options)).
We recommend Clerk for the easiest setup:

<Tabs>
  <Tab title="Clerk (Recommended)">
    Install Clerk and configure "Sign in with ARES":

    ```bash
    npm install @clerk/nextjs
    ```

    Add Clerk environment variables to `.env`:

    ```bash
    NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=your_clerk_publishable_key
    CLERK_SECRET_KEY=your_clerk_secret_key
    ```

    Next, follow [this guide](/authentication/clerk#step-3:-configure-ares-as-oauth-provider-in-clerk) to configure ARES as a custom OAuth provider in your Clerk Dashboard

  </Tab>

  <Tab title="Direct OAuth Token">
    If you already have an OAuth access token from another source:

    ```typescript
    import { Ares } from 'ares-sdk';

    const client = new Ares({
      accessToken: 'your_oauth_access_token',
    });
    ```

    [Custom OAuth guide →](/authentication/custom-oauth)

  </Tab>
</Tabs>

## Step 4: Create ARES API route

Create a complete ARES integration with **one line of code** using the SDK helpers:

<CodeGroup>

```typescript Next.js + Clerk (Recommended)
// app/api/ares/route.ts
import { createAresRoutes } from "ares-sdk/lib/api-routes";

// That's it! Full ARES integration in 1 line ✨
export const { GET, POST } = createAresRoutes();
```

```typescript Manual Setup (Alternative)
// app/api/ares/user/route.ts
import { Ares } from "ares-sdk";
import { auth } from "@clerk/nextjs/server";
import { createAresFromClerkAuth } from "ares-sdk/lib/clerk";
import { clerkClient } from "@clerk/nextjs/server";

export async function GET() {
  // Get ARES client from Clerk auth
  const aresClient = await createAresFromClerkAuth(auth(), {
    clerkClient: clerkClient(),
  });

  if (!aresClient) {
    return Response.json({ error: "Not authenticated" }, { status: 401 });
  }

  // Retrieve user information
  const user = await aresClient.user.retrieve();

  return Response.json({
    name: user.name,
    email: user.email,
    credits: user.credits,
  });
}
```

```typescript Node.js Express
// routes/ares.js
import express from "express";
import { Ares } from "ares-sdk";

const router = express.Router();

router.get("/user", async (req, res) => {
  // Get OAuth token from your auth middleware
  const accessToken = req.user.aresToken;

  const client = new Ares({ accessToken });

  try {
    const user = await client.user.retrieve();
    res.json({
      name: user.name,
      email: user.email,
      credits: user.credits,
    });
  } catch (error) {
    res.status(500).json({ error: error.message });
  }
});

export default router;
```

</CodeGroup>

**What this creates:**

- `GET /api/ares` - Returns user's ARES profile and credit balance
- `POST /api/ares` - Records usage and deducts credits
- Auto-handles authentication, errors, and validation

## Step 5: Add client-side UI component

Use the React hook to display credits in your UI:

```tsx
// components/credits-counter.tsx
"use client";

import { useAres } from "ares-sdk/lib/react";

export function CreditsCounter() {
  const { user, loading, error } = useAres("/api/ares", {
    refreshInterval: 300000, // Refresh every 5 minutes
    revalidateOnFocus: true, // Refresh on window focus
  });

  return (
    <div>
      <h2>{user.credits.toLocaleString()} Credits</h2> // User's credit balance
      <p>{user.name}</p>
    </div>
  );
}
```

**Features included:**

- ✅ Auto-fetches user data on mount
- ✅ Auto-refreshes periodically
- ✅ Auto-refreshes on window focus
- ✅ Built-in loading and error states
- ✅ Full TypeScript support

## Step 6: Charge user credits

Add credit billing to any API route - the SDK handles balance checks and deductions for you:

```typescript
// app/api/generate-content/route.ts
import { withCreditsCheck } from "ares-sdk/lib/api-routes";

export const POST = withCreditsCheck(
  {
    credits: 50, // Cost in credits
    usage: "AI content generation",
  },
  async (request, { user, body }) => {
    const result = await generateContent(body.prompt);
    return Response.json({ result });
  }
);
```

**The SDK handles everything for you:**

- Parses the request body automatically
- Checks user has enough credits
- Deducts credits before your code runs
- Returns 402 error if insufficient credits
- Gives you a typed body in your handler

<Note>
  No manual credit checking needed - just specify the cost and the SDK does the
  rest.
</Note>
